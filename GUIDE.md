# 📖 InfraStack คืออะไร — คู่มือฉบับคนไม่รู้อะไรเลย

> เอกสารนี้เขียนสำหรับคนที่ได้ repo นี้ไปแล้วสงสัยว่า "มันคืออะไร ใช้ยังไง ทำอะไรได้บ้าง"
> ไม่ต้องมีพื้นฐาน DevOps มาก่อน — ถ้าพร้อมลงมือติดตั้งจริง ไปต่อที่ [docs/00-introduction.md](docs/00-introduction.md)

## 1. มันคืออะไร (อธิบายใน 1 ย่อหน้า)

InfraStack คือ **ชุดติดตั้งเซิร์ฟเวอร์สำเร็จรูป** — ปกติถ้าคุณเช่าเซิร์ฟเวอร์ (VPS) มาเปล่าๆ
คุณต้องนั่งติดตั้งของเป็นสิบตัวเอง: ตัวรันเว็บ, ฐานข้อมูล, ระบบ HTTPS, ระบบสำรองข้อมูล,
ระบบเฝ้าดูว่าเว็บล่มไหม ฯลฯ ใช้เวลาหลายวันและพลาดง่าย repo นี้ย่อทั้งหมดนั้นเหลือ:

```bash
git clone <repo นี้>
cd infra-stack
./install.sh
```

ประมาณ 10–15 นาที ได้เซิร์ฟเวอร์พร้อมใช้ระดับ production — และทุกเครื่องที่ติดตั้ง
ด้วยชุดนี้จะมีหน้าตาเหมือนกันหมด ดูแลง่าย ย้ายเครื่องง่าย

**เปรียบเทียบ:** VPS เปล่า = ที่ดินเปล่า / InfraStack = บ้านสำเร็จรูปพร้อมระบบไฟ ประปา
กล้องวงจรปิด และแปลนต่อเติม — คุณแค่ยกเฟอร์นิเจอร์ (เว็บของคุณ) เข้าไปวาง

## 2. ต้องมีอะไรก่อนใช้

| ต้องมี | คือ | หาได้จาก |
|--------|-----|----------|
| VPS | เซิร์ฟเวอร์เช่ารายเดือน/รายปี (Ubuntu, แนะนำ 4 CPU / 8GB RAM ขึ้นไป) | Contabo, Hetzner, GreenCloud, DigitalOcean ฯลฯ |
| โดเมน | ชื่อเว็บ เช่น `example.com` | Namecheap, Cloudflare, GoDaddy ฯลฯ |
| Cloudflare account (ฟรี) | ใช้จัดการ DNS + กันโจมตี (แนะนำแต่ไม่บังคับ) | cloudflare.com |
| ความกล้าพิมพ์คำสั่งใน terminal | copy-paste เป็นก็พอ คู่มือมีคำสั่งให้ทุกขั้น | — |

## 3. ข้างในมีอะไรบ้าง (อธิบายทีละตัวแบบภาษาคน)

### 🏗️ โครงสร้างพื้นฐาน — ทุกอย่างวิ่งผ่านพวกนี้

| ตัว | มันคือ | เปรียบเหมือน |
|-----|--------|--------------|
| **Docker** | ตัวรันโปรแกรมแบบ "กล่อง" แยกกัน — แต่ละแอปอยู่กล่องใครกล่องมัน พังไม่ลามกัน ลบง่าย ย้ายง่าย | ตู้คอนเทนเนอร์บนเรือสินค้า |
| **Traefik** | ประตูหน้าบ้าน — รับทุก request จากอินเทอร์เน็ต แล้วส่งต่อไปยังแอปที่ถูกต้องตามชื่อโดเมน แถมทำ **HTTPS (กุญแจเขียว) ให้อัตโนมัติฟรี** | พนักงานต้อนรับที่รู้ว่าแขกมาหาใคร |
| **PostgreSQL** | ฐานข้อมูลหลัก — ที่เก็บข้อมูลถาวรของทุกแอป (users, orders, posts...) | ตู้เอกสารของบริษัท |
| **Redis** | หน่วยความจำเร็วจัด — เก็บของชั่วคราว (cache), คิวงาน, session login | โพสต์อิทแปะหน้าจอ + กล่อง inbox งาน |

### 🖥️ หน้าจอจัดการ — เปิดผ่านเบราว์เซอร์ ไม่ต้องพิมพ์คำสั่ง

| ตัว | เปิดที่ | ใช้ทำอะไร |
|-----|---------|-----------|
| **Portainer** | `portainer.<โดเมน>` | เห็น/สั่ง restart/ดู log ทุกแอปในเครื่อง — ศูนย์บัญชาการหลัก |
| **pgAdmin** | `pgadmin.<โดเมน>` | เปิดดู/แก้ฐานข้อมูล PostgreSQL แบบมี GUI (เหมือน phpMyAdmin) |
| **Redis Insight** | `redis.<โดเมน>` | ส่องข้างใน Redis ว่ามี cache/คิวอะไรอยู่ |

### 📊 ระบบเฝ้าระวัง — รู้ก่อนลูกค้าว่ามีปัญหา

| ตัว | เปิดที่ | ตอบคำถามว่า |
|-----|---------|-------------|
| **Netdata** | `netdata.<โดเมน>` | "**ตอนนี้**เครื่องเป็นอะไร" — CPU/RAM/Disk แบบวินาทีต่อวินาที |
| **Grafana** (+Prometheus) | `grafana.<โดเมน>` | "**เมื่อวาน/อาทิตย์ก่อน**เกิดอะไร" — กราฟย้อนหลัง 30 วัน |
| **Uptime Kuma** | `uptime.<โดเมน>` | "เว็บไหนล่มบ้าง" — เช็คทุกนาที + **แจ้งเตือนเข้า Telegram** |
| **Dozzle** | `logs.<โดเมน>` | "แอปตัวนี้ error อะไร" — ดู log สดๆ ผ่านเบราว์เซอร์ |
| **Loki** | (ดูผ่าน Grafana) | "3 วันก่อน error อะไร" — ค้น log ย้อนหลังได้ |
| **Alertmanager** | (ทำงานเบื้องหลัง) | ยาม — CPU/RAM/Disk เกินลิมิต → ส่งข้อความเข้า Telegram เอง |
| **Watchtower** | (ทำงานเบื้องหลัง) | อัปเดตโปรแกรมให้อัตโนมัติทุกตี 4 (เฉพาะตัวที่เราอนุญาต) |
| **Beszel** | `servers.<โดเมน>` | มีหลายเซิร์ฟเวอร์ → ดูทุกเครื่องจากจอเดียว |

### 📦 ของเสริม (มีให้ แต่ยังไม่เปิดจนกว่าจะต้องใช้)

| ตัว | ใช้เมื่อ |
|-----|----------|
| **RabbitMQ** | คิวงานซับซ้อน — งาน AI หลายขั้นตอน, retry, กระจายงานหลาย worker |
| **MinIO** | อยากมีที่เก็บไฟล์แบบ S3 เป็นของตัวเอง (ปกติแนะนำใช้ Cloudflare R2 ก่อน) |

## 4. โครงโฟลเดอร์ — อะไรอยู่ตรงไหน

```text
infra-stack/
├── install.sh        ← จุดเริ่มต้น รันตัวนี้บนเครื่องใหม่
├── TODO.md           ← checklist งานหลังติดตั้ง
├── docs/             ← คู่มือทุกเรื่อง เรียงเป็นบท 00–25 (เริ่มที่ 00)
├── services/         ← "พิมพ์เขียว" ของแต่ละโปรแกรมข้างบน (1 ตัว = 1 โฟลเดอร์)
├── docker/           ← ตัวรวมพิมพ์เขียวเป็นชุดๆ (core / database / monitoring / ...)
├── scripts/          ← คำสั่งสำเร็จรูป: ติดตั้ง, backup, deploy, ย้อนเวอร์ชัน
├── templates/        ← แม่แบบสำหรับสร้างเว็บใหม่ (laravel / nextjs / n8n / ai-worker)
├── projects/         ← เว็บจริงของคุณจะมาอยู่ตรงนี้ (สร้างจาก templates)
├── backup/           ← สคริปต์สำรองข้อมูล + ที่เก็บไฟล์ backup
├── monitoring/       ← ค่าตั้งของระบบเฝ้าระวัง + เกณฑ์แจ้งเตือน
└── environments/     ← ตัวอย่างค่าตั้งต่อสภาพแวดล้อม (production / staging / dev)
```

หลักการเดียวที่ต้องรู้: **1 โปรแกรม = 1 โฟลเดอร์ใน services/** อยากรู้อะไรตั้งค่ายังไง
เปิดโฟลเดอร์มัน มี README ของตัวเองทุกตัว

## 5. ใช้งานยังไง — 3 เรื่องที่ทำบ่อยสุด

### 5.1 ติดตั้งเครื่องใหม่ครั้งแรก

```text
1. เช่า VPS (Ubuntu) → ได้ IP มา
2. ชี้ DNS: สร้าง A record (portainer, grafana, netdata, uptime, logs) → IP เครื่อง
3. ssh เข้าเครื่อง → clone repo → sudo ./scripts/harden.sh   (ล็อกประตูบ้านก่อน)
4. ./install.sh → แก้ .env (ใส่โดเมน + ตั้งรหัสผ่าน) → เสร็จ
5. เปิด docker/docker-compose.database.yml และ .monitoring.yml ตามคู่มือ
6. ไล่ทำ TODO.md
```

เวอร์ชันละเอียดทีละคลิก: [docs/00-introduction.md](docs/00-introduction.md) → ไล่ตามบท
(บทที่คนพลาดบ่อยมีหัวข้อ "🩹 ปัญหาที่เจอจริง" บอกทางแก้ไว้แล้ว)

### 5.2 เอาเว็บขึ้น (deploy)

```bash
# สร้างจากแม่แบบ — laravel / nextjs / n8n / ai-worker
./scripts/create-project.sh myapp laravel

# ใส่ค่าจริง (โดเมน, รหัสฐานข้อมูล ฯลฯ)
nano projects/myapp/.env

# ขึ้นเว็บ
cd projects/myapp && docker compose up -d
```

เพิ่ม DNS record `myapp.<โดเมน>` → เปิดเว็บได้เลย มี HTTPS ให้อัตโนมัติ

**deploy ครั้งถัดๆ ไป** ใช้คำสั่งเดียว — ระบบจะเปิดเวอร์ชันใหม่คู่กับตัวเก่า
รอจนแน่ใจว่าใช้งานได้ ค่อยสลับ (เว็บไม่ดับเลยแม้แต่วินาทีเดียว และถ้าเวอร์ชันใหม่พัง
เว็บเดิมยังอยู่):

```bash
./scripts/deploy.sh myapp        # deploy แบบเว็บไม่ดับ
./scripts/rollback.sh myapp v1.2 # เวอร์ชันใหม่มีปัญหา? ถอยกลับใน 1 นาที
```

### 5.3 ดูแลรายวัน/รายสัปดาห์

| อยากรู้ว่า... | ทำแบบนี้ |
|---------------|----------|
| ทุกอย่างยังปกติไหม | ไม่ต้องทำอะไร — ถ้าผิดปกติ Telegram จะเด้งเอง |
| แอปตัวไหน error | เปิด `logs.<โดเมน>` (Dozzle) คลิกดูตัวนั้น |
| ทำไมเครื่องช้า | เปิด `netdata.<โดเมน>` ดู CPU/RAM ตอนนี้ |
| อยาก restart แอป | เปิด `portainer.<โดเมน>` → Containers → เลือก → Restart |
| backup ยังทำงานไหม | `ls -lh backup/archives/postgres/` (ต้องมีไฟล์ใหม่ทุกวัน) |

Checklist สุขภาพประจำสัปดาห์ (5 นาที): [docs/14-monitoring.md](docs/14-monitoring.md)

## 6. ทำอะไรได้บ้าง (use cases จริง)

- ✅ โฮสต์เว็บ **Laravel / Next.js** หลายเว็บบนเครื่องเดียว แยกกันเด็ดขาด
- ✅ รัน **n8n** ทำ automation — auto post โซเชียล, AI chatbot, เชื่อม API ต่างๆ
- ✅ รัน **AI worker** หลังบ้าน — สร้างรูป/คลิป/ประมวลผลโดยเว็บหน้าบ้านไม่สะดุด
- ✅ **CI/CD ฟรี** — push โค้ดเข้า GitHub แล้วเว็บอัปเดตเองใน 2-4 นาที (ไม่ใช้ GitHub Actions ไม่มีค่าใช้จ่าย)
- ✅ Deploy แบบ **zero-downtime** — อัปเดตเว็บโดยผู้ใช้ไม่รู้สึกอะไรเลย
- ✅ **Backup อัตโนมัติทุกคืน** + ส่งสำเนาขึ้น cloud (Cloudflare R2)
- ✅ เครื่องพัง? เช่าเครื่องใหม่ → clone → install → restore backup → กลับมาใน ~30 นาที
- ✅ โตขึ้น? มีแผน scale เป็นหลายเครื่องโดยไม่ต้องรื้อ ([docs/18-scale.md](docs/18-scale.md))

## 7. กติกาความปลอดภัย (สั้นๆ แต่ห้ามข้าม)

1. **ห้าม commit ไฟล์ `.env` หรือรหัสผ่าน/token ใดๆ เข้า Git** — repo นี้ตั้ง `.gitignore` กันไว้แล้ว อย่าไปฝืนมัน
2. ค่าที่เขียนว่า `CHANGE_ME` ทุกจุด = ต้องเปลี่ยนก่อนใช้จริง (สร้างรหัสด้วย `openssl rand -hex 24`)
3. ถ้าใช้กับงานจริง ตั้ง repo เป็น **Private**
4. เข้าเซิร์ฟเวอร์ด้วย SSH key เท่านั้น (ตัวติดตั้ง `harden.sh` จัดให้) — เก็บไฟล์ key ไว้ให้ดี หายแล้วเข้าเครื่องไม่ได้
5. Backup ที่ไม่เคยลอง restore = ไม่มี backup — ซ้อมเดือนละครั้ง
